## False Positives Eliminated

| Original Finding | Why It's Not a Bug |
|-----------------|-------------------|
| Sandbox type duplication (`SandboxType`, `ExecEnv`, `SandboxTransformError`) | **Adapter pattern.** `handlers/sandboxing.rs` defines simplified types with explicit `From` conversions to canonical types. Intentional architecture, not duplication. |
| `AST_GREP_OVERRIDE` duplication in `ast_grep_binary.rs` vs `semantic.rs` | **Independent statics.** Two separate `Lazy<Mutex<...>>` instances serving different code paths (`resolve_ast_grep_binary_from_env_and_fs` vs `resolve_ast_grep_binary`). Tests for each path use their own override. |
| `render/mod.rs:24` `strip_ansi_codes` | **Re-export, not duplicate.** One-liner delegating to `text_utils::strip_ansi_codes`. |
| Model constants in provider modules | **Provider-scoped pattern.** Each provider defines constants for models it supports. Cross-cutting models get defined in each referencing module. Maintenance risk only, not a bug. |
| `result_large_err` suppressions needing boxing fix | **Already mitigated.** `LLMErrorMetadata` is already `Box<LLMErrorMetadata>` at `vtcode-commons/src/llm.rs:598`. Enum is ~40 bytes, under the 64-byte threshold. Suppressions are stale, not the underlying issue. |
